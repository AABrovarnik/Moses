## 2. Карточки моделей

### 2.1. Kimi K2.7-code:cloud

- **Что это:** развитие линейки Moonshot Kimi (K1.5 → K2). K2.7 — гипотетическая версия, **в открытых источниках на июнь 2026 г. подтверждения нет** (K2 был released 2025, K2.5/Thinking-варианты упоминались в roadmap).
- **Архитектура (по семейству):** MoE, ~1T total / 32B active параметров (по K2).
- **Контекст:** ~128K–256K (требует проверки для K2.7).
- **Сильные стороны:** длинный контекст, китайский язык, длинный chain-of-thought, agentic tool use.
- **Цена:** неизвестна (требует проверки).
- **РФ-доступ:** через посредника (Moonshot AI — китайский провайдер, прямых платежей из РФ нет).
- **Когда выбирать:** если подтвердится — длинный контекст, китайские/английские задачи, альтернатива Qwen3-Coder.
- **Действие:** **проверить актуальный модельный ряд Moonshot на сайте перед использованием**.

---

### 2.2. GLM-5.2:cloud

- **Что это:** Zhipu AI (THUDM/智谱AI). Подтверждённая линейка: GLM-4.5 (июль 2025), 4.5-Air, 4.6 (сентябрь 2025, 200K контекст), 4.6V (мультимодальный), 4.7 (конец 2025). **GLM-5.2 в открытых источниках на июнь 2026 г. явно не описан** (есть GLM-5 как «следующее поколение, начало 2026», но конкретно 5.2 — требует проверки).
- **Архитектура (по семейству):** MoE, 355B/32B active (GLM-4.5), 106B/12B active (GLM-4.5-Air).
- **Контекст:** 200K (по 4.6).
- **Сильные стороны:** кодинг (SWE-bench, Terminal-Bench, multi-file), tool use, длинные agentic-задачи, китайский язык, MIT-лицензия для open-source версий.
- **Цена:** pay-as-you-go, в 5–10 раз дешевле западных моделей (по 4.5/4.6).
- **РФ-доступ:** через посредника, API открыт.
- **Когда выбирать:** production coding-агент, где Qwen3-Coder не хватает, и нужна альтернатива западным моделям.
- **Действие:** **проверить актуальный модельный ряд Zhipu перед использованием**.

### 2.3. MiniMax-M3:cloud

- **Что это:** модель от MiniMax (КНР/Сингапур). **В открытых источниках на июнь 2026 г. подтверждённой информации нет** (запросы WebSearch не вернули данных; имя в скриншоте похоже на опечатку или внутреннее обозначение провайдера).
- **Контекст:** неизвестен.
- **Сильные стороны:** неизвестны.
- **Цена:** неизвестна.
- **РФ-доступ:** ⚠️ — уточнять.
- **Когда выбирать:** **не выбирать без подтверждения**. Это «чёрный ящик» из скриншота.
- **Действие:** **обязательная проверка источника** (сайт провайдера, Hugging Face, документация). Если модель реальна — карточку дополнить.

---

### 2.4. Nemotron-3-Super:cloud

- **Что это:** NVIDIA Nemotron-3-Super. **Подтверждённая модель**, релиз — 11 марта 2026 г.
- **Архитектура:** 120B total / 12B active параметров MoE hybrid Mamba-Transformer, LatentMoE, MTP (Multi-Token Prediction), NVFP4 pretraining.
- **Контекст:** **1M токенов** — на уровне Claude Opus 4.7 / Sonnet 4.6.
- **Производительность (бенчмарки):**
  - LiveCodeBench v5: **81.19** (опережает GPT-OSS-120B, чуть ниже Qwen3.5-122B)
  - SWE-Bench (OpenHands): **60.47**
  - HMMT Feb25: **93.67** (лучший в своём классе)
  - AIME25: **90.21**
  - GPQA: **79.23** (82.70 с tools)
  - Terminal Bench Core 2.0: 31.00
- **Пропускная способность:** до 2.2× GPT-OSS-120B, 7.5× Qwen3.5-122B (8k in / 64k out на B200).
- **Open release:** BF16, FP8, NVFP4 чекпоинты + датасеты под NVIDIA Nemotron Open Model License.
- **Доступность:** build.nvidia.com, Hugging Face, Perplexity, OpenRouter, vLLM/SGLang/TRT-LLM, партнёры: Google Cloud, OCI, CoreWeave, Together AI, Fireworks.
- **Цена:** требует уточнения (cloud NVIDIA-партнёров).
- **РФ-доступ:** ⚠️ через посредника.
- **Когда выбирать:** agentic reasoning-задачи с длинным контекстом, on-prem (open weights), где нужна альтернатива Anthropic/OpenAI.
- **Действие:** реальный кандидат на production-стек, проверить тарифы NVIDIA-партнёров.

---

### 2.5. Gemma 4 (Google DeepMind)

- **Что это:** Gemma 4 — линейка open-source моделей Google. Релиз — 2 апреля 2026 г., лицензия Apache 2.0.
- **Семейство (5 размеров):**
  - E2B: 2.3B effective (5.1B с embeddings), 128K контекст
  - E4B: 4.5B effective (8B), 128K
  - **12B Unified:** 11.95B dense, 256K
  - **26B A4B:** 26B MoE (4B active), 256K
  - **31B dense:** 31B, 256K
- **Ключевые фичи:**
  - **Мультимодальность** (текст, изображения, видео; аудио — в E2B/E4B)
  - **PLE** (Per-Layer Embeddings) — новая архитектура, у каждого decoder-слоя свой embedding
  - **Shared KV Cache** — оптимизация памяти для длинного контекста
  - **Нативный system prompt** с thinking/reasoning режимами
  - **Function calling** для agentic workflows
  - **140+ языков**
- **Позже (июнь 2026):** QAT-чекпоинты — квантизация при обучении, E2B влезает в 1 ГБ RAM.
- **Производительность:**
  - 31B: MMLU Pro 85.2%, AIME 2026 89.2%, Codeforces ELO **2150**
  - 26B MoE: LMArena 1441 при 4B active
- **Деплой:** transformers, llama.cpp, MLX, transformers.js, Mistral.rs, vLLM, SGLang, Unsloth, **Ollama, LM Studio** (day-0 поддержка).
- **Цена:** **$0** (open weights, локально).
- **РФ-доступ:** ✅ on-prem, без ограничений.
- **Когда выбирать:** on-prem агент, где критична лицензия Apache 2.0, мультимодальность, длинный контекст. Сильная альтернатива Qwen3-Coder для нерусскоязычных задач.

---

### 2.6. Qwen 3.6

- **Что это:** эволюция линейки Qwen3 от Alibaba. В моих обзорах [vibe-coding](2026-06-17-vibe-coding-obzor.md) и [сводном](2026-06-17-ai-stack-svodny-obzor.md) фигурирует **Qwen3-Coder** как стратегический выбор для РФ-проектов.
- **Уточнение:** **Qwen 3.6** в моих источниках на июнь 2026 г. — не подтверждённая отдельная модель. Возможные интерпретации:
  1. Qwen3-Coder следующего поколения (семейство Qwen3-Coder активно обновляется: 30B-A3B, 480B-cloud).
  2. Внутреннее/неподтверждённое имя.
  3. Ошибка OCR/написания.
- **Что известно по семейству Qwen3-Coder:**
  - 30B-A3B-Instruct: 30B total / 3B active, 256K контекст, MoE
  - 480B-cloud: облачная версия
  - Сильный в SWE-bench, agentic coding, tool use
  - Open weights, можно локально через Ollama
- **РФ-доступ:** ✅ on-prem (через Ollama) + облачный API Alibaba Cloud.
- **Когда выбирать:** основной on-prem coding-agent в РФ-стеке. Замена или альтернатива Sonnet 4.6 для задач, где не нужна западная модель.
- **Действие:** **проверить актуальное имя у Alibaba** (qwen3-coder, qwen3-coder-next, или qwen3.6 — что вышло в 2026 г.).

---

### 2.7. Qwen 2.5:7b

- **Что это:** Qwen 2.5, dense 7B. Семейство выпущено в 2024–2025 г., открытые веса.
- **Контекст:** 128K.
- **Сильные стороны:** компактность, низкие требования (≈5 ГБ VRAM в Q4), поддержка русского/английского, function calling.
- **Слабые стороны:** уступает Qwen3-Coder-30B в coding-задачах; старше Qwen3.
- **Цена:** **$0** (локально через Ollama).
- **РФ-доступ:** ✅ on-prem, без ограничений.
- **Когда выбирать:** лёгкий fallback, edge-устройства, CI/CD пайплайн с автодополнением, прототипы. Не для production coding-agent.
- **Альтернатива:** Qwen3:4b (чуть новее архитектура) при тех же требованиях.

---

### 2.8. Qwen 3:4b

- **Что это:** Qwen3, dense 4B. Семейство Qwen3 — 2025 г. (плотные: 0.6B, 1.7B, 4B, 8B, 14B, 32B; MoE: 30B-A3B, 235B-A22B).
- **Контекст:** 128K.
- **Сильные стороны:** компактность, нативный thinking mode, function calling, поддержка 100+ языков.
- **Слабые стороны:** 4B параметров — мало для серьёзного coding-agent, качество кода заметно ниже 30B+.
- **Цена:** **$0** (локально через Ollama).
- **РФ-доступ:** ✅ on-prem, без ограничений.
- **Когда выбирать:** самый лёгкий on-prem fallback, edge/браузер (через transformers.js), демо. Не для production.

## 4. Практические рекомендации по интеграции в Ollama-стек

Из [сводного обзора стеков](2026-06-17-ai-stack-svodny-obzor.md) рекомендованный паттерн — **Ollama-runtime** как единый endpoint.

```bash
# Локальные модели (on-prem)
ollama pull qwen2.5:7b
ollama pull qwen3:4b
ollama pull gemma4:31b          # когда появится в Ollama-библиотеке
ollama pull qwen3-coder:30b     # основной coding-agent

# Cloud-модели через Ollama-как-прокси (настроить через litellm или прямой OpenAI-compatible endpoint)
# Base URL: http://localhost:11434/v1
# В Cline / LangGraph / Claude Agent указывается прокси
```

**Важно:** для `:cloud`-эндпоинтов нужен API-ключ провайдера. Прямого платежа из РФ нет ни у одного из 8 провайдеров — только через посредника или корпоративный контракт.

## 5. Контрольный список перед выбором

```
[ ] 1. Модель реально существует? (проверить на сайте провайдера / Hugging Face)
[ ] 2. Версия актуальна? (не alpha/legacy)
[ ] 3. Open weights или проприетарная?
[ ] 4. Контекст соответствует задаче? (≥32K для coding-agent, ≥128K для long-horizon)
[ ] 5. Tool use / function calling поддерживается?
[ ] 6. Лицензия совместима с РФ-деплоем? (Apache 2.0 / MIT — без ограничений)
[ ] 7. Цена: $/1M in/out для cloud, GPU-требования для local
[ ] 8. Бенчмарк на целевой задаче (кодинг: SWE-bench, LiveCodeBench)
[ ] 9. On-prem возможен? (если compliance — обязательно)
[ ] 10. Vendor lock-in оценён? (есть ли fallback на другую модель)
```

**Если 2+ пунктов «нет данных» — не берите в production.**
