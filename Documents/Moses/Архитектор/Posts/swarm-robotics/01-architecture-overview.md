# Глава 1. Базовые архитектуры управления роем

## Что такое рой

Рой (swarm) — группа автономных агентов, координирующихся для общей цели без жёсткой иерархии. От классической мультиагентной системы рой отличает:
- децентрализация принятия решений,
- локальное взаимодействие (агент видит только соседей),
- эмерджентное глобальное поведение,
- масштабируемость (от 10 до 100 000 единиц без изменения архитектуры).

## Три основные архитектуры

### 1. Централизованная (master/worker)

Один мастер отдаёт команды, остальные исполняют.

**Кто главный:** выделенный мастер-агент или стационарный сервер.

**При гибели мастера:**
- hot standby — переключение за 10–100 мс
- переизбрание (Bully/Raft) — секунды-минуты простоя
- в простых системах — полная остановка (недопустимо для критичных миссий)

**При потере части агентов:** производительность падает линейно, но задача продолжает решаться, пока жив мастер.

**Где применяют:** Kiva/Amazon Robotics (склад), Kubernetes control plane, оркестрация контейнеров.

**Плюсы:** предсказуемость, простота отладки, глобальный оптимум.
**Минусы:** единая точка отказа, плохая масштабируемость, требует стабильного канала связи.

### 2. Децентрализованная («истинный рой»)

Нет постоянного лидера. Каждый агент автономен, опирается на свои сенсоры и обмен с ближайшими соседями. Глобальное поведение возникает (emerges) из локальных правил.

**Кто главный:** формально — никто. Фактически — все одновременно. Иногда выделяется временный лидер под задачу (dynamic leadership).

**При гибели любого агента:** рой не замечает. Агенты продолжают работать по локальным правилам.

**При потере части агентов — graceful degradation:**
- 100% → 80% — потеря почти не сказывается
- 80% → 40% — производительность падает на 30–50%
- 40% → 10% — ниже рабочего порога для большинства задач
- 10% → 0% — для простых задач (удержание формации) ещё может работать

**Критическая масса** проектируется значительно ниже номинального размера. 500 — это не минимум, а запас прочности и скорости.

**Где применяют:** военные рои (Perdix, LOCUST), воздушные шоу (Verity Studios, UVify), колонии муравьёв, стаи птиц.

**Плюсы:** нет единой точки отказа, лёгкое масштабирование, устойчивость.
**Минусы:** сложно проектировать и отлаживать, поведение иногда неинтуитивно, локальный оптимум ≠ глобальный.

### 3. Иерархическая

Агенты объединены в кластеры с сублидерами. Сублидеры ↔ сублидеры — по децентрализованным принципам, сублидер ↔ агенты группы — по централизованным.

**Кто главный:** иерархия сублидеров. На верхнем уровне — координатор кластеров.

**При гибели лидера кластера:** сублидер переизбирается за секунды, группа продолжает работу.

**Где применяют:** крупные логистические системы, военные формирования, edge-облачные вычисления.

**Плюсы:** баланс предсказуемости и устойчивости, лучшая масштабируемость чем у чисто централизованной.
**Минусы:** сложнее обоих предыдущих вариантов, риск каскадного отказа.

## Алгоритмы выбора лидера (если он нужен)

| Алгоритм | Суть | Применение |
|---|---|---|
| Bully (1982) | Агент с наивысшим ID инициирует выборы | Простые системы |
| Ring-based | Токен по кругу | Критичные системы |
| Raft (2014) | Консенсус с лидером и журналом | Kubernetes, etcd |
| Paxos (1989) | Консенсус через кворум | Распределённые БД |
| Эмерджентный | Лидер «проявляется» из локальных взаимодействий | Биоинспирированные рои |

В роевой робототехнике чаще используют эмерджентный выбор: агент с наибольшим ресурсом (энергия, вычислительная мощность, лучшая позиция) становится центром притяжения. Явного голосования нет.

## Модели поведения агентов

| Модель | Сложность | Примеры |
|---|---|---|
| Реактивная (стимул-ответ) | Минимальная | Боты-пылесосы, ant-following |
| Конечный автомат (FSM) | Средняя | Промышленные роботы, дроны-курьеры |
| Вероятностная (Markov) | Средняя | Ant Colony Optimization |
| Нейросетевая (политика) | Высокая | Boston Dynamics, обученные дроны |
| LLM-агенты (2024–2026) | Высокая | AutoGPT, CrewAI, мультиагентные фреймворки |

Самые распространённые в железе — реактивные и FSM. LLM-агенты активны в софтверных роях.

## Ключевые алгоритмы

- **Ant Colony Optimization (ACO)** — Dorigo, 1992. Агенты-муравьи оставляют «феромоны» на хороших путях.
- **Particle Swarm Optimization (PSO)** — Kennedy & Eberhart, 1995. Агенты-частицы «летят» в сторону лучших найденных решений.
- **Boids (Reynolds, 1987)** — три правила для стаи: держаться рядом, согласовать скорость, двигаться к центру.
- **Consensus algorithms** — все агенты соглашаются с состоянием (используется в роевых сенсорных сетях).
- **Stigmergy** — координация через изменения в среде (феромоны — частный случай).
