# Глава 3. IoT-рой и сенсорные сети

## Что такое IoT-рой

Масса мелких устройств с ограниченной энергией, которые собирают данные и координируются без центрального сервера (или с минимальным его участием). Характерные черты:
- сотни–десятки тысяч узлов,
- узлы работают от батарейки месяцы и годы,
- каждый узел «тупой» (микроконтроллер, а не полноценный CPU),
- связь — низкоскоростная, ближняя (LoRa, Zigbee, BLE, Thread, Matter).

## Архитектура по умолчанию: иерархическая

В чистом виде децентрализованный IoT-рой — редкость, потому что:
- лимит по памяти и CPU у каждого узла (типично 32–256 КБ RAM, 1–100 МГц),
- нужна энергоэффективность (лишние пересылки едят батарею),
- нужен «шлюз» для интеграции с облаком или человеком.

Поэтому классическая схема:
```
[конечные узлы] → [mesh-сеть узлов-ретрансляторов] → [шлюз/edge] → [облако]
```

Но: внутри mesh-сети поведение часто роевое. Узлы сами выбирают, кто будет ретранслятором, кто уснёт для экономии энергии, кто проснётся, чтобы опросить соседей.

## Ключевые алгоритмы

### Энергоэффективность
- **Duty cycling** — узел спит 99% времени, просыпается на миллисекунды раз в несколько секунд
- **Data aggregation** — узлы не передают сырые данные, а объединяют (среднее, медиана, max) по пути к шлюзу
- **Clustering** — узлы образуют кластеры с «главой» (cluster head), которая агрегирует данные кластера
- **Predictive sampling** — если данные предсказуемы (например, температура), узел спит дольше

### Самоорганизация mesh
- **RPL (Routing Protocol for Low-Power and Lossy Networks)** — стандарт IETF для IoT-mesh. Строит дерево маршрутов от узлов к шлюзу, но с автоматическим перестроением при отказе узла
- **Zigbee mesh** — проприетарный, но широко распространённый (Philips Hue, IKEA TRÅDFRI)
- **Thread + Matter** — современный стек, IPv6 в mesh, обратная совместимость с Zigbee через мосты
- **BLE mesh** — для носимых устройств и proximity-приложений
- **LoRaWAN** — не mesh, а звезда (узел → шлюз). Дальность до 15 км, но без роевых свойств

### Роевое поведение
- **Gossip protocols** — каждый узел раз в N секунд «шепчет» случайному соседу. Через O(log N) раундов все знают всё. Применяется для синхронизации состояний
- **Consensus** — узлы соглашаются с общим значением (например, «температура в зоне X = 24.3°C»). Используется для устранения шумных датчиков
- **Ant Colony Optimization** — для маршрутизации в mesh
- **Token-based coordination** — для последовательного опроса (энергоэффективнее, чем broadcast)

## Протоколы верхнего уровня

| Протокол | Модель | Энергия | Где |
|---|---|---|---|
| **MQTT** | pub/sub, нужен брокер | Средняя | IoT общего назначения |
| **CoAP** | REST-подобный, поверх UDP | Низкая | Ограниченные устройства |
| **AMQP** | очередь сообщений | Высокая | Промышленный IoT |
| **DDS** | pub/sub без брокера, peer-to-peer | Высокая | Беспилотники, V2X, военные |
| **LwM2M** | управление устройствами | Низкая | Телеметрия SIM-карт |

## Реальные системы

### Умный дом
- **Philips Hue** — Zigbee mesh, шлюз Hue Bridge, до 50 ламп. Роевые свойства: каждая лампа ретранслирует сигнал (mesh)
- **IKEA TRÅDFRI** — Zigbee, шлюз DIRIGERA
- **Matter (CSA, 2022–н.в.)** — попытка объединить экосистемы. Thread для mesh, Wi-Fi/Ethernet для бэкбона. К 2026 году — основной стандарт умного дома
- **Apple HomeKit** — не mesh, хаб-архитектура
- **Google Home** — аналогично, но с Thread Border Router в Nest Hub

### Промышленный IoT (IIoT)
- **WirelessHART** — стандарт ISA100.11a, mesh из датчиков на заводах
- **ISA100 Wireless** — конкурент WirelessHART
- **OPC UA over TSN** — детерминированная сеть реального времени для заводов
- **5G private networks** — срез сети для завода, поддержка миллионов устройств на км²

### Умный город
- **Уличное освещение с mesh** — каждая лампа — узел mesh. При отказе одной лампы соседние компенсируют яркость
- **Парковки** — датчики в каждом месте, mesh в каждую зону
- **Умные счётчики** — электричество, вода, газ. Mesh между счётчиками, передача показаний в конце месяца
- **Trafﬁc systems** — датчики трафика, mesh, адаптивные светофоры

### Сельское хозяйство
- **Vineyard sensors** — датчики влажности, температуры по всему полю, mesh, edge-аналитика
- **Smart greenhouses** — тысячи датчиков на гектар
- **Livestock tracking** — BLE-метки на животных, mesh-антенны на ферме

### Экология и научные применения
- **Зеленый проект (Great Duck Island)** — 150 датчиков на острове, мониторинг птиц, mesh
- **Корневые системы растений** — датчики влажности и pH в почве, mesh
- **Вулканы** — датчики сейсмической активности
- **Ледники** — датчики движения льда в Арктике
- **Коррозия мостов и трубопроводов** — mesh из датчиков на конструкции

## Кто главный в IoT-рое

Классический IoT-рой — **иерархический**:
- **Cluster head** в mesh — не лидер, а «курьер». Ротируется каждые N часов или при разряде батареи
- **Шлюз** — единственная точка выхода в облако. Если шлюз упал — mesh всё ещё работает, данные копятся
- **Облако** — аналитика, ML, long-term storage. Если облако недоступно — edge продолжает работать

В более продвинутых системах (например, AWS IoT Greengrass, Azure IoT Edge) **edge сам принимает решения** — облако нужно только для долговременной аналитики. Это уже ближе к полной децентрализации.

## Что если часть узлов выбыла?

- **Шлюз отказал:** mesh перестраивается, данные копятся. Когда шлюз вернётся — выгружаются. Потеря — часы, не дни
- **Cluster head отказал:** новый выбирается за секунды (RPL/аналоги)
- **10–20% узлов отказали:** покрытие ухудшается, но данные собираются. Если датчики избыточны (3–5 на зону) — точность почти не падает
- **50% узлов отказали:** критическая масса для многих задач. Mesh может развалиться на острова (островная проблема)
- **90% узлов отказали:** mesh не работает. Остатки работают точечно

## Проблемы и ограничения

### Безопасность
- Узлы дешёвые, шифрование слабое
- Один скомпрометированный узел = дыра в mesh
- **Решения:** аппаратные root of trust (TPM/SE), secure boot, прошивка по воздуху (OTA) с подписью
- **Реальные инциденты:** Mirai botnet (2016) — 600 000 IoT-устройств как DDoS-оружие

### Стандарты
- Z-Wave, Zigbee, Thread, Matter, HomeKit, Alexa Smart Home, Google Home — 6+ экосистем
- Matter пытается объединить, но медленно
- 6LoWPAN, RPL, CoAP, MQTT-SN — академические, но работающие

### Энергия
- **Батарея:** CR2032 = 230 мАч. Активный режим с радио = 10–30 мА. Значит, ~10 часов активной работы, или 1–2 года в duty cycle 0.1%
- **Energy harvesting:** солнечные панели на узле, вибрация, разница температур — дают милливатты
- **Backscatter:** узел отражает радиоволны считывателя, не имеет собственного передатчика. Почти нулевая энергия
